# hidong_2048 PRD

## 1. 项目目标
在保留 `hidong_2048` 原有视觉风格基础上，升级为一个可直接构建、可部署到 Cloudflare Pages 的完整网页游戏项目。

项目重点不是做成普通 2048 换皮，而是做成“收集四位嗨咚豆豆英雄”的主题版本。

---

## 2. 核心玩法

### 2.1 基础规则
- 保持经典 2048 的移动与合并规则
- 玩家通过上下左右移动棋盘推动数字方块
- 相同数字方块碰撞后合并为更高数值

### 2.2 英雄生成规则
- 每个英雄有独立的目标值配置
- 可选目标值：`128 / 256 / 512`
- 默认值：四个英雄默认都为 `128`
- 当普通数字方块合并到某个英雄的目标值时，会生成对应英雄 tile

### 2.3 多英雄同目标值逻辑
- 若多个英雄配置成同一个目标值（例如都为 `128`）
- 每次合成出该值时，应从“该目标值下尚未收集的英雄”中按顺序生成下一个
- 已经收集过的英雄不能被重复生成
- 当该目标值下的英雄全部收集完成后，再次合成该值时，只保留为普通数字 tile

### 2.4 英雄 tile 规则
- 英雄 tile 一旦生成，不再参与任何合并
- 英雄 tile 仍然占据棋盘格子
- 英雄 tile 可随整体滑动被推动移动
- 英雄 tile 不会升级为更高数字

### 2.5 通关条件
- 同一局中收集到四种不同英雄，即判定通关

### 2.6 配置生效时机
- 英雄目标值配置修改后，只对“新开一局”生效
- 不在当前对局中途动态改规则，以避免状态不一致

## 3. 界面要求

### 3.1 桌面端
- 桌面端优先展示完整信息
- 英雄图鉴上移，优先展示
- 标题缩小，减少头部占高
- 统计区更像 dashboard
- 右侧展示当前局的分数、步数、用时、预估结算等信息
- 配置面板不常驻页面，改为右上角 `⚙️` 按钮打开弹窗
- 游戏介绍与规则说明放到页面下方

### 3.2 移动端
- 不采用简单压缩桌面版的方案
- 使用同一页面下的独立移动 layout
- 主界面尽量做到一屏可玩，不需要明显上下滚动
- 主屏保留：
  - 标题
  - 核心状态
  - 棋盘
  - 新开一局
  - 英雄进度
- 次要内容收纳进右上角菜单/抽屉
- 为适配较矮屏幕，顶部统计与英雄条合并并压缩
